02

Scripts Overview

PlayerController.cs

  • Rigidbody AddForce movement
  • Reads FloatingJoystick input
  • Camera-relative movement direction
  • Slow Slerp rotation (Pummel-style)
  • Keyboard fallback (WASD/Space)
  • Jump via button or keyboard

CameraFollow.cs

  • Smooth Lerp position follow
  • Smooth Slerp rotation look-at
  • Configurable offset & speed
  • Runs in LateUpdate (smooth)

KnockbackObject.cs

  • Detects player collision
  • Applies impulse force away from player
  • Red color flash on hit
  • Attach to any Rigidbody cube

FloatingJoystick.cs

  • Spawns at touch position
  • Fully self-contained (no packages)
  • Dead zone support
  • Horizontal + Vertical output

Open the Project in Unity

⚠️ You need Unity 2022 LTS (2022.3.x). Download it free from unity.com/download via Unity Hub.
1
Copy the project folder to your computer

Unzip UnityMobileGame.zip to any location, e.g. C:\Projects\UnityMobileGame

2
Open Unity Hub

Launch Unity Hub. On the left sidebar click Projects.

3
Add the project

Click the Add button (top right) → Add project from disk → navigate to the UnityMobileGame folder → click Open.

4
Make sure Unity version is 2022.3.x

In Unity Hub, the project entry shows the Unity version. If it says "Version mismatch", click the version dropdown and select your installed 2022.3 version.

5
Click Open / Launch

Unity will compile scripts. This takes 1–3 minutes on first open. Wait until the progress bar in the bottom right finishes.

04

Auto-Build the Scene (One Click!)

🔧 The SceneSetup.cs script adds a menu item to Unity that builds the entire scene automatically — ground, player, camera, cubes, joystick UI, all wired up. Run it once.
1
Create a new empty scene

In Unity: File → New Scene → select Basic (Built-in) → click Create.

2
Make sure the Assets/Scenes folder exists

In the Project window (bottom), right-click Assets → Create → Folder → name it Scenes. Also create a folder named Materials.

3
Run the setup tool

In the Unity top menu bar: click Tools → Setup Game Scene. Watch the Scene view fill up!

4
Check the Console window

Open Window → General → Console. You should see: "SceneSetup: DONE! Scene saved to Assets/Scenes/GameScene.unity"

💡 If you don't see the Tools menu, Unity hasn't compiled the scripts yet. Wait for the spinning loading icon in the bottom right to stop.

Wire Up the Jump Button

The Jump Button's OnClick event needs to be linked manually because Unity's editor-scripted UnityEvents don't serialize automatically. This takes 30 seconds.

1
Find the JumpButton in Hierarchy

In the Hierarchy panel (left side): GameCanvas → JumpButton. Click it.

2
Find the Button component in Inspector

In the Inspector (right side), scroll down to the Button component. You'll see an On Click () section at the bottom.

3
Click the + button

At the bottom of the On Click section, click the + icon. A new row appears.

4
Drag Player to the object slot

From the Hierarchy, drag the Player GameObject into the empty object field (where it says "None (Object)").

5
Select the method

Click the function dropdown (says "No Function"). Choose PlayerController → OnJumpButtonPressed ().

Save the scene

2

Build APK for Android

⚠️ You need Android Build Support installed in Unity Hub. Open Unity Hub → Installs → click the gear ⚙️ next to your Unity 2022 version → Add Modules → check Android Build Support + Android SDK & NDK Tools + OpenJDK.
1
Switch platform to Android

File → Build Settings. In the platform list click Android, then click Switch Platform. Wait for re-import (may take a few minutes).

2
Add scene to build

In Build Settings, click Add Open Scenes. Make sure Assets/Scenes/GameScene.unity is listed with ✅.

3
Configure Player Settings

Click Player Settings button. Set Company Name, Product Name, and Package Name (e.g. com.yourname.pummelparty). Set Minimum API Level to Android 7.0 (API 24).

4
Build!

Back in Build Settings, click Build. Choose a folder (e.g. Builds/Android). Unity generates a .apk file. This takes 2–10 minutes.

5
Install on phone

Copy the .apk to your Android phone. Enable Install Unknown Apps in phone settings. Tap the APK to install.

Zip the Project

📦 Before zipping, delete the Library/, Temp/, and Logs/ folders — they are auto-generated by Unity and can be 2–10 GB in size. The zip will be much smaller without them.
1
Close Unity

Close Unity completely before zipping to avoid file locks.

2
Delete cache folders

In File Explorer / Finder, open the UnityMobileGame folder and delete these folders: Library/   Temp/   Logs/   obj/

3
Zip the folder

Windows: Right-click UnityMobileGame → Send to → Compressed (zipped) folder.
Mac: Right-click → Compress "UnityMobileGame".

Terminal (alternative — faster) 
# Navigate to parent folder of your project
cd C:\Projects   # or /Users/yourname/Projects on Mac

# Zip (exclude Library, Temp, Logs)
zip -r UnityMobileGame.zip UnityMobileGame \
    --exclude "*/Library/*" \
    --exclude "*/Temp/*" \
    --exclude "*/Logs/*" \
    --exclude "*/.git/*"

Troubleshooting

!
"Tools" menu doesn't appear

Wait for Unity to finish compiling. Check the bottom-right progress bar. If there are errors in Console, fix them first. The SceneSetup.cs script is editor-only (#if UNITY_EDITOR) so it won't appear in builds.

!
Player falls through the ground

Make sure the Ground plane has a Collider (it does by default). Also check the Player's Rigidbody is not set to IsKinematic.

!
Joystick doesn't work in Play mode

Make sure the Canvas has a GraphicRaycaster component and there's an EventSystem in the Hierarchy. SceneSetup creates both automatically.

!
Player rotates too fast / slow

Select Player in Hierarchy → Inspector → PlayerController → adjust Rotation Speed. Lower = more Pummel Party-style clumsy turning.

!
Player tips over

The PlayerController automatically freezes X and Z rotation. If this somehow got reset, select Player → Rigidbody component → Constraints → check Freeze Rotation X and Z.

!
APK build fails

Make sure Android Build Support module is installed (Unity Hub → Installs → gear icon). Also check that JDK/SDK paths are set in Edit → Preferences → External Tools.
